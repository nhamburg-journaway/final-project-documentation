es eine gute Planung. In diesem Abschnitt wird daher aufgezeigt, welche Ressourcen (Zeit, Hardware)
wie eingesetzt werden.

\subsection{Abweichungen vom Projektantrag}
\label{sec:AbweichungenProjektantrag}
Bei dem Inhalt des Projektes wird die Anbindung von Hangfire mit der Anbindung von RabbitMQ ausgetauscht, 
da bei der Planung erkannt wurde, dass Hangfire in der ersten Iteration nicht benötigt wird und sich bei der 
Umsetzung der Event Queues auf den Message Broker RabbitMQ geeinigt wurde.

\subsection{Ressourcenplanung}
\label{sec:Ressourcenplanung}
\textbf{Hardware}
\begin{enumerate}
	\item Büroarbeitsplatz
	\item Netzwerk mit Internetzugang
	\item Laptop und zwei weitere Bildschirme
	\item Server für die Anwendung und Datenbank  
\end{enumerate}
\textbf{Software}
\begin{enumerate}
	\item Betriebssystem Laptop: Windows 11
	\item Entwicklungsumgebung: JetBrains Rider 2026
	\item Message Broker Software: RabbitMQ
	\item Datenbanksoftware: PostgreSQL 16
	\item Zielbetriebssystem: Linux Debian 12.13
	\item Software zum generieren und einsehen von Testdaten auf der Datenbank: HeidiSQL & pgAdmin 4  
\end{enumerate}
\textbf{Personal}
\begin{enumerate}
	\item Für die Qualitätssicherung während der Review- und Testphase werden zwei weitere Entwickler benötigt.  
\end{enumerate}

\subsection{Entwicklungsprozess}
\label{sec:Entwicklungsprozess}
Während der Umsetzung des Projektes wird ein agiler Prozess bzw. Scrum verfolgt. 
Dies ist ein Prozess, bei dem iterativ kleinere Abschnitte eines Auftrages fertiggestellt werden,
wodurch schnell auf unvorhergesehene Probleme oder neue Anforderungen eingegangen werden kann.
Scrum zeichnet sich durch eine hohe Flexibilität aus und eignet sich gut für kleine Projekte bzw. Teams.
